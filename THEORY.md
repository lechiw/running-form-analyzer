# 🧠 跑姿分析，到底在分析什么？

> 一个视频扔进去，怎么就知道你跑得好不好？
> 这篇用大白话把背后的原理讲清楚。

---

## 一、我们在干什么

简单说就是三步：

```
视频 → 找到你的关节位置 → 算出各种跑姿数据 → 打分 + 给建议
```

**关键问题：** 摄像头拍的是 2D 画面，人站在不同距离、不同角度。怎么保证不管你站近站远，算出来的数字都一样？

答案是：**尽量用角度，不用绝对距离。**

---

## 二、怎么找到你的关节？

用的是 Google 的 **MediaPipe Pose**。

它本质上是一个经过大量训练（上百万张带标注的人体图片）的深度学习模型。你给我一张图，它给你 33 个点的坐标：

```
  0. 鼻子
  11. 左肩 ─────── 12. 右肩
   |                   |
  13. 左肘             14. 右肘
   |                   |
  15. 左腕             16. 右腕
   |
  23. 左髋 ─────── 24. 右髋
   |                   |
  25. 左膝             26. 右膝
   |                   |
  27. 左踝             28. 右踝
   |                   |
  29. 左足跟           30. 右足跟
```

每个点带三个信息：**x（横向）、y（纵向）、visibility（置信度，这个点我有多大把握找对了）**。

> **精度提示：** MediaPipe 在侧面视角下精度最高（正面视角下左右会混）。所以我们一直建议横屏侧面拍，就是这个原因。

---

## 三、核心思路：尺子会变，角度不变

### 为什么这是个问题？

同样的跑步动作，你站远处拍和站近处拍，在画面里的像素数完全不一样：

```
远处拍：躯干 100 像素        近处拍：躯干 300 像素
步频 170 spm                步频 170 spm
```

步频用 spm（步/分钟）不受影响，但**垂直振幅**如果直接用像素值，近处拍的"振幅"会比远处拍的大好几倍——即使你跳得一样高。

### 我们的解法：每帧算一把"尺子"

**假设：** 成人的躯干（髋→肩）大约 50 厘米。

于是每帧都算：
```
这帧的尺子 = 50 厘米 ÷ 画面里躯干的像素数
```

近处：躯干 300px → 1px ≈ 0.17cm
远处：躯干 150px → 1px ≈ 0.33cm

垂直振幅从 40px → 40 × 0.17 = 6.8cm，不管远近都算出同一个数字。

> ⚠️ 这个假设不是精确的（有人腿长有人腿短），但对**同一个人前后对比**来说足够了。误差通常在 10-20% 以内。

### 角度型指标：完全不依赖尺子

角度这东西有个好处：**跟距离无关**。

```
肘关节 90°
┌────┐
│    │
└────┘

近拍显示 90°         远拍显示 90°
一样。
```

所以躯干前倾角、摆臂幅度（肘关节角度范围）、膝关节角度——这些直接用**余弦定理**算，跟你在画面里占大占小没关系。

## 四、每个指标到底在测什么？（用打比方说）

### 1️⃣ 步频（Cadence）— "你的腿在敲鼓"

```
定义：一分钟脚落地的次数
理想：170-180 spm
```

**怎么算的：** 跑起来时，髋关节会上下颠。我们检测这个"颠簸"的频率——颠一次，踩一步。用**自相关**算法（就是问"这个信号跟自己错位后有多像"）找到最稳定的步频。

**为什么 170-180 最好？**

把跑步想象成**弹跳球**：
- 步频太低（<160 spm）→ 脚在空中太久 → 落地时速度大 → 冲击力大
- 步频太高（>190 spm）→ 步子太小 → 效率低
- 170-180 刚好卡在**自然谐振频率**上——身体用最少的能量维持速度

**现实表现：**
- 基普乔格：185 spm
- 大迫杰：195-200 spm（他个子小，用高步频弥补）
- 业余跑者：平均 150-165 spm

---

### 2️⃣ 躯干前倾（Trunk Lean）— "你的上半身在栽葱吗？"

```
定义：脊梁骨离垂直方向歪了多少度
理想：4-10° 微微前倾
```

**怎么算的：** 肩的中点 → 髋的中点，画一条线，看它跟竖直方向夹角多大。

**为什么这个角度重要？**

把身体想象成一根**倒立的杆子**：
- 站太直（<3°）：重心在后，每一步都在"刹车"，膝盖压力大
- 前倾太多（>15°）：腰需要用力拉住身体，下背痛风险高
- 4-10° 微微前倾：重心刚好在脚的前面，地心引力在"拉"你往前跑——省力

**有意思的细节：** 很多人以为跑步是用腿跑，其实上身的姿态比腿更重要。躯干前倾 4° 的跑者，比 0° 的跑者，每一步省力约 3%。一场马拉松算下来，总共省的能量够多跑 2 公里。

---

### 3️⃣ 垂直振幅（Vertical Oscillation）— "你在跳，还是在跑？"

```
定义：跑步时身体上下颠的幅度
理想：<10 cm
```

**怎么算的：** 把髋关节每帧的高度画成一条曲线。用二次曲线去掉摄像头抖动（手持拍摄的上下晃动），然后看剩下的"真正的跑步颠簸"有多大。用**4×标准差**而不是最大最小值——防止某一步的异常颠簸污染结果。

**为什么越小越好？**

把身体想象成一个**弹簧**：
- 颠 10cm 的时候，那 10cm 就是**浪费的力**——你花了力气往上弹，但你要的是往前走
- 每次颠 10cm → 每公里多消耗约 5% 的能量
- 精英马拉松跑者垂直振幅通常在 6-8cm

**但注意：** 完全没有也不行。跑步本身就是利用肌腱的弹性势能（像弹簧蓄能释放），振幅为零说明你不会利用这个弹性。

---

### 4️⃣ 手臂对称性（Arm Symmetry）— "你的左右手在吵架吗？"

```
定义：两只手摆臂动作有多对称
理想：>80/100（越高越好）
```

**怎么算的：** 分别看左右肘关节的最大角度 - 最小角度（摆臂范围）。如果差的太多，说明身体在用躯干旋转代偿。

**为什么对称性重要？**

人的腿和手臂是**交叉耦合**的：
```
左臂往前 → 右腿往前（正常的跑步姿态）
```

如果你左臂摆得比右臂小很多，躯干会不自觉地旋转来补偿——长期下来，一边的髋关节和膝盖会承受不对称的压力。

**简单自测：** 下次跑步时感觉一下。如果跑完发现一边肩膀比另一边酸，大概率是对称性有问题。

---

### 5️⃣ 触地距离与着地方式 — "你的脚落在哪？"

```
触地距离定义：脚跟和髋关节的水平距离
理想：<15 厘米
着地方式：前掌 / 中掌 / 后跟
```

**怎么算的：** 找到离髋最远的那只脚的脚跟，量它跟髋的 x 方向距离。用前面说的尺子（px_to_cm）换算成真实厘米。

**为什么触地距离要短？**

想象你在一辆车里，一脚踩在地上想让自己停下来：

```
过度跨步（>15cm）:            正常步幅（<15cm）:
    🏃                        🏃
     \_                        _/
       ↘                     ↙
        o                      o
    刹车效应                  流畅推进
```

脚落在身体前方 → 地面对你施加"向后"的力 → 每一步都在微微刹车 → 每一步都得再加速。这非常费劲，而且对膝盖是冲击。

**着地方式：**
- **前掌着地：** 跑得快、效率高、膝盖友好，但小腿压力大
- **中掌着地：** 折中，对大多数人来说是最优解
- **后跟着地：** 休闲跑者最常见，膝盖冲击最大。但也不是完全不可以——如果你跑得慢（配速 <6min/km），后跟着地挺舒服的

---

### 6️⃣ 髋部倾斜（Hip Drop）— "你的骨盆歪了没？"

```
定义：跑步时左右髋关节的高度差
理想：<2 厘米
```

**怎么算的：** 每一帧看左髋和右髋的垂直高度差。

**为什么这个重要？**

想象一张**餐桌**——四个脚一样高就稳，一个脚短了就晃。

跑步时单脚支撑的时候，如果臀中肌力量不够，骨盆会往抬起来的那只脚的方向下坠。久而久之：
- 髂胫束综合征（ITBS）— 跑者膝盖外侧痛的最常见原因
- 下背痛
- 膝盖内侧压力增大

这是个**跑者最容易忽视但又最常见的伤病根源**。

---

### 7️⃣ 触地时间（Ground Contact Time）— "你的脚在地上烫多久？"

```
定义：跑步时脚从落地到离地的时间
理想：<250 毫秒
```

**怎么算的：** 检测"脚靠近地"到"脚离开地"之间经过的帧数 × 帧时长。

**为什么越短越好？**

脚在地上每多待一毫秒，地面反作用力就多传一毫秒到你的骨骼。触地时间短 = 弹性好 = 效率高。

```
精英跑者：     180-200 ms
业余跑者：     220-300 ms
走路：         ~400 ms
```

## 五、跑姿评分是怎么打出来的？

### 老方法：阶梯函数（这次换掉了）

以前是：
```
步频 168 → 80 分
步频 166 → 60 分    ← 差 2spm 掉 20 分？不合理
```

### 新方法：Sigmoid 平滑曲线

```
步频 175 → 100 分 （精准命中理想值）
步频 170 → 82 分   （略偏，轻微扣分）
步频 165 → 50 分   （偏离较多，合理扣分）
步频 155 → 18 分   （严重偏离）
```

像正态分布曲线——中间高两边低，平滑过渡。现实世界中，人体表现的衰退本来就是渐变的，不是"一脚踩空"式。

### 各指标权重

```
步频          20%   ← 最基础的
躯干前倾      18%
垂直振幅      18%
触地控制      17%
髋部倾斜      10%
手臂对称      10%
触地时间       7%    ← 还在调试中，权重低
```

### 一个有意思的细节

我们没有对所有指标一视同仁。**着地方式**（前掌/中掌/后跟）不直接扣分，而是**浮动加分**：

- 中掌/前掌着地 → +3 分（奖励）
- 后跟着地 → 不加分也不扣分（很多人后跟着地也跑得挺好的，只是要小心）

这样避免了一刀切的判断。

## 六、这套方法的局限性（坦诚版）

### 做得好的
✅ **同一个人前后对比** — 例如练了两周后再拍一次，看步频有没有提高
✅ **趋势分析** — 疲劳分析看前段后段的差异
✅ **相对偏差检测** — 手臂是不是一边多一边少，骨盆有没有歪
✅ **大的问题捕捉** — 过度跨步、振幅太大、前倾过度

### 做不好/需要注意的
❌ **不同跑者精确横向比** — MediaPipe 的骨架精度本身就有几像素的误差
❌ **正面/背面视角** — 只有侧面视角的数据可靠
❌ **衣服太宽松** — MediaPipe 依赖轮廓，穿宽大 T 恤会降低精度
❌ **复杂光线** — 逆光、过曝会导致检测失败
❌ **多人画面** — 画面里有别人？MediaPipe 只检测检测到的第一个人

### 最推荐的使用方式

```
1. 同一台手机，同一个机位
2. 侧面拍，横屏，全身入画
3. 第一次跑 → 分析 → 针对弱项练一周
4. 同一机位再拍一次 → 看指标有没有改善
5. 重复
```

而不是：
```
❌ 拿自己的分析结果跟基普乔格的数据比
❌ 正面拍的视频扔进来期待能用
```

## 七、写在最后

跑步是一件非常**个人化**的事情。

这套工具不是为了告诉你"你跑得不对"，而是为了让你看到自己看不见的东西——因为你跑步的时候不可能一边跑一边看自己的后脚跟落在哪、骨盆有没有歪。

**数据是镜子，不是裁判。**

你用镜子整理衣领，挺好。但别一直盯着镜子跑。

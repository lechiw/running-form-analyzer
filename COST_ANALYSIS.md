# 💰 部署成本分析

> 重点分析：视频上传/下载带宽成本、计算成本、存储成本
> 数据基于实际测试：15s 视频 → 6.3MB 预览，30s 视频 → 预估 12MB

---

## 一、单次分析的花费拆解

每次用户上传一个跑步视频，经历以下流程：

```
用户上传（39MB）→ 存储 → 骨架提取 → 指标计算 → 可视化渲染 → 用户下载结果
                                                                  ├── HTML 报告（~50KB）
                                                                  └── 预览视频（~6-25MB）
```

### 1.1 传输流量（按次计）

| 环节 | 数据量 | 方向 | 成本敏感度 |
|------|:-----:|:----:|:---------:|
| 用户上传原始视频 | **~39 MB** | 入站 | 🟢 很多云服务商**入站免费** |
| 用户下载 HTML 报告 | **~50 KB** | 出站 | 🟢 忽略不计 |
| 用户下载预览视频（15s） | **~6 MB** | 出站 | 🟡 中等 |
| 用户下载预览视频（30s） | **~12 MB** | 出站 | 🟡 中等 |
| 用户下载预览视频（60s） | **~25 MB** | 出站 | 🔴 高 |

**结论：视频下载是主要带宽成本。** 上传通常免费，报告极小，但预览视频每个用户每次都要完整下载。

### 1.2 计算成本（按次计）

| 环节 | 耗时 | CPU | 备注 |
|------|:----:|:---:|------|
| MediaPipe 骨架提取 | 10-30s/分钟视频 | 1-2 核 | 纯 CPU，不耗 GPU |
| 指标计算 | <1s | 几乎不计 | 纯 NumPy 计算 |
| ffmpeg 渲染 + 重编码 | 10-30s/分钟视频 | 1-2 核 | 最耗 CPU 的部分 |
| LLM 报告（可选） | 1-3s | API 调用 | 消耗 DeepSeek/OpenAI API |

**结论：** 一段 30s 视频大约需要 **20-60 秒 CPU 时间**（1 核）。这是计算成本的主体。

### 1.3 LLM API 成本（可选）

如果用 DeepSeek（推荐，便宜）：

| 模型 | 每次分析 Token 数 | 单次成本 |
|------|:---------------:|:--------:|
| DeepSeek Chat | ~2000 tokens（输入）+ ~1000（输出） | **~¥0.001** |
| OpenAI GPT-4o-mini | 同上 | **~¥0.003** |
| OpenAI GPT-4o | 同上 | **~¥0.03** |

**结论：LLM 成本几乎可以忽略。** 一个月分析 1000 次也才几块钱。

## 二、部署方案对比

### 方案 A：阿里云/腾讯云 VPS（推荐给 1-10 人用）

**配置推荐：**

```
2 vCPU + 4GB RAM  + 40GB SSD  +  5Mbps 带宽
```

| 项目 | 价格 | 说明 |
|------|:----:|:----|
| 服务器 | **¥99/月** | 2C4G 轻量云，新用户首年更便宜 |
| 带宽 | **包在 ¥99 里** | 5Mbps ≈ 每月可传约 **1000-2000 次** 30s 视频 |
| 超出带宽 | ¥0.8/GB | 按量计费，每 1000 次额外约 ¥12 |
| 存储 | ¥0.3/GB/月 | 40GB 够存 1000 条分析结果 |
| 域名 | ¥30-50/年 | 可选，IP 直访也行 |
| SSL 证书 | 免费 | Let's Encrypt |

**每月总成本：¥100-150**

**单次分析成本（1000 次/月）：**
```
带宽：99元包月已含约 200GB 流量 → 够用
计算：服务器闲着也是闲着，算上 = 0
存储：40GB 循环利用 → 可忽略
LLM：¥0.001 × 1000 = ¥1
─────────────────────────
平均每次成本：≈ ¥0.15
```

**瓶颈：带宽。** 5Mbps ≈ 每秒传 640KB。一个 12MB 的预览视频需要约 **19 秒**才能下载完。如果并发 3 个人同时下载，排队会更慢。

**解决方案：**
- 上传 CDN（如阿里云 OSS + CDN）：带宽成本降到 ¥0.18/GB
- 限制视频时长（建议拍 15-20s 足够分析）

---

### 方案 B：对象存储 + Serverless（适合 10-100 人，成本更低）

架构：用户上传到 OSS → 触发函数计算 → 结果存 OSS → 用户从 OSS 下载

| 项目 | 价格 | 说明 |
|------|:----:|:----|
| 对象存储 | **¥0.12/GB/月** | 阿里云 OSS，按量付费 |
| 函数计算 | **¥0.0011/次** | 每次分析约 20-60s CPU 时间 |
| CDN 出站 | **¥0.18/GB** | 视频下载走 CDN |
| 上传流量 | **免费** | OSS 上传不收费 |

**每月成本（100 次/月）：**
```
存储：100 × 25MB（临时）= 2.5GB → ¥0.3/月
计算：100 × ¥0.02（60s CPU）= ¥2/月
CDN：100 × 12MB × ¥0.18/GB = ¥21.6/月
─────────────────────────
总共：≈ ¥24/月
```

**每月成本（1000 次/月）：**
```
存储：1000 × 25MB ≈ 25GB → ¥3/月
计算：1000 × ¥0.02 = ¥20/月
CDN：1000 × 12MB × ¥0.18/GB = ¥216/月
─────────────────────────
总共：≈ ¥240/月
```

**瓶颈仍然是出站带宽。** 达到 1000 次/月后，CDN 出站费用占 90% 以上。

## 三、视频流量到底占多少？

**这是一个关键数据：** 一次完整分析的用户体验流量 = 用户上传 + 用户下载

假设用户上传 30s 视频（~25MB），下载 12MB 预览：

```
上行（免费）: 25MB
下行（收费）: 12MB
────────────────
合计账单流量: 12MB
```

**瓶颈锚点：出站流量总价**

| 月分析次数 | 出站流量 | CDN 费用（¥0.18/GB） | 服务器方案费用 |
|:---------:|:--------:|:------------------:|:------------:|
| 10 | 120 MB | **¥0.02** | ¥99 包月（浪费） |
| 100 | 1.2 GB | **¥0.22** | ¥99 包月（值得） |
| 500 | 6 GB | **¥1.08** | ¥99 包月（划算） |
| 1000 | 12 GB | **¥2.16** | ¥99 包月（非常划算） |
| 5000 | 60 GB | **¥10.80** | ¥99 包月（够用） |
| 10000 | 120 GB | **¥21.60** | ¥99 包月 + ¥20+ |

**结论：** 
- **< 500 次/月：** VPS ¥99 包月最划算，带宽都包在里头了
- **500-5000 次/月：** VPS 仍然最划算，5Mbps 带宽足够
- **> 5000 次/月：** 考虑 OSS + CDN 或自建 CDN 节点

## 四、省钱技巧

### 技巧 1：压缩预览视频（已在做）

当前预览视频已经是 H.264 re-encode（从 19MB _raw.mp4 压到 6.3MB）。但还可以更省：

```
当前：H.264, 3500 kbps, 15s → 6.3MB
优化：H.265 (hevc), 1500 kbps, 15s → 2.8MB  ← 省 55%
```

**代价：** H.265 编码慢约 2-3 倍，但下载体积减半。如果带宽是瓶颈，值得。

### 技巧 2：不渲染视频

用户可以选择**不生成可视化视频**，只拿 HTML 报告 + 数字指标。一个纯文本报告只有 **50KB**，几乎不花流量。

这是**最有效的省钱方式**——直接去掉 99% 的下载流量。

### 技巧 3：视频有效期自动删除

设置存储自动过期：

```
视频上传 → 分析完成后保留 24 小时 → 自动删除
```

这样存储成本可以忽略。用户分析完马上下载结果就行。

### 技巧 4：限制上传大小

前端限制上传 ≤ 100MB/视频。30 秒跑步视频通常在 15-40MB，足够分析。超过 100MB 的大概率是拍太长或者拍完没裁剪。

### 技巧 5：用 DeepSeek 不用 OpenAI

```
DeepSeek Chat: ¥0.001/次
OpenAI GPT-4o: ¥0.03/次
────────────────
差 30 倍
```

如果 1000 次/月，DeepSeek 花 ¥1，4o 花 ¥30。

## 六、结论：核心成本是"视频下载"，不是"视频分析"

```
每次分析费用（1000次/月）：
  CPU 计算：    ¥0.02 ← 几乎不计
  LLM 报告：    ¥0.001 ← 几乎不计
  存储：        ¥0.003 ← 几乎不计
  视频下载：    ¥0.08-0.22 ← 占 80-95%
```

**如果你要省成本，只有一个方向值得花精力：**
→ **减少预览视频的下载体积**（编码压缩、降低码率、或者让用户自行选择是否下载视频）

或者换一种说法：

> **跑步分析本身很便宜，贵的是把带骨架的视频发给用户看。**
> 如果只给数字指标 + 文字报告，成本直接降到接近 0。
